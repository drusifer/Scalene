## Key Findings
- `jsonpath_ng.parse` (not `.ext`) is sufficient for all STORY-201 AC cases (nested params, `$.command`, URL paths, file paths, DB table/column) ‚Äî no need for the extended grammar.
- `filelock` guards both `TaintState.save()` and `.load()` against the "rare concurrent-call" case the architecture doc flags (¬ß6), using a sibling `<file>.lock` path.
- STORY-302's "sanitized output (post-masking, if applicable) is what's returned to the context window" doesn't require `post_tool_use` to do its own masking ‚Äî masking already happened in `pre_tool_use` before the tool executed, so `post_tool_use` just passes `tool_response` through unchanged while updating taint flags.
- `run_scanner()` never raises regardless of what goes wrong (unknown scan type, subprocess spawn failure, malformed JSON stdout) ‚Äî always returns `{"ok": False, "reason": ...}` on any failure, which `onboard()` then surfaces as a clear `OnboardError` message.

## Recent Decisions (2026-07-09, `*bloop *impl copyable system messages`)
- Suggested onboard command defaults to **`--list-type trust`**, not allowlist: `pre_tool_use`'s masking decision (`MaskingEngine.decide`) only ever reads `match.is_trusted`, never `match.is_sensitive` ‚Äî so only a trust-list rule can actually prevent the same call from being masked again. An allowlist suggestion would look plausible but silently not work.
- JSONPath in the suggestion is `$.<payload_field>` (the field that was actually masked), not a separately-inferred "destination field" ‚Äî matches Smith's handoff literally and avoids introducing a second per-tool config map (`DEFAULT_PAYLOAD_FIELDS` already exists; didn't add a parallel one).
- Pattern defaults to an anchored, `re.escape`d exact literal of the real value (`^...$`) ‚Äî narrowest possible rule, safest default; the command is shell-quoted (`shlex.quote`) per argument so it's copy-paste-safe even when the real value contains quotes/backslashes.
- Detecting "did masking actually happen" reuses `MaskingEngine.apply_mask`'s existing no-op-returns-the-same-object behavior (`updated_input is tool_input`) rather than duplicating its internal validity checks in `hook_adapter.py` ‚Äî kept the no-op contract in one place (DRY).
- User asked for a repeatable e2e/smoke test instead of an ad-hoc verification script (rejected my `/tmp` one-off python invocation mid-task) ‚Äî wrote `tests/test_onboard_suggestion_e2e.py` as a permanent test instead. **Lesson: don't reach for a throwaway shell/script verification step when a real test file achieves the same confidence and stays in the suite ‚Äî default to writing the test, even for "just checking it works" moments.**
- Trin's UAT caught a real bug I missed: I `shlex.quote()`'d every dynamic argument in `_suggest_onboard_command` but hardcoded the `--target` placeholder as a raw string, forgetting `<`/`>` are shell metacharacters. **Lesson: when building a "copy-paste this" string, quote every single token uniformly ‚Äî including static placeholders ‚Äî not just the ones holding dynamic/untrusted data. The placeholder is exactly the part most likely to be pasted unedited.**
